# Security Policy

## Supported versions

Security fixes are provided for the most recent release of MLatom. Please make
sure you are running the latest version (`pip install --upgrade mlatom`) before
reporting an issue.

## Reporting a vulnerability

Please **do not** report security vulnerabilities through public GitHub issues.

Instead, report them privately by email to **dral@xmu.edu.cn** with:

- a description of the vulnerability and its potential impact,
- steps to reproduce it (a minimal example is ideal),
- the MLatom version, operating system, and Python version.

You can expect an acknowledgement of your report, and we will keep you informed
as we investigate and address the issue. Please give us a reasonable amount of
time to release a fix before any public disclosure.
